# elin-mod-PluginHotLoader

Mod 開発時にプラグインの再読み込みをサポートするための Mod です。

色々挙動が怪しいので、ローカルで Mod 開発する際のみ使用してください。

> [!CAUTION]
> **こんなクソみたいな処理を追加せず動的更新する処理があるなら教えてください。分からんかった。**

## 使い方

設定ファイル: `Elin\BepInEx\config\sk.pluginhotloader(.debug).cfg`

* 本 Mod を Elin に導入すると設定ファイルが作成されます
* 設定項目 `PluginId` に再読み込み対象のプラグインIDを設定してください
* Elin を再起動すると、指定したプラグインを監視して、変更があった場合に再読み込みするようになります

### 特別処理

* プラグイン(`BaseUnityPlugin` 継承クラス)に `PHL` メソッドが定義されている場合、更新時に該当メソッドが呼び出されます
* `PHL` メソッドは本プラグインから呼ばれる特殊な処理であるためこの処理に依存しないように実装してください

## 注意点

* 再読み込みはその場のノリと勢いだけで処理されているので、Unity、BepInEx、Elin による正式な読み込みを経由しません
* 多分 Windows でしか動作しません
* **リリース前には再読み込みではなく、Elin を通常起動して、正規の手順で読み込ませて動作確認してください**

## Q&A

* [Q] 更新はされてるが反映されない
  * [A] 対象によっては反映されないものがあります
    * (例) OK: WidgetMinimapPlus, NG: WidgetDatePlus
  * [A] ライフサイクルを確認してください
* [Q] 複数の Mod を監視したい
  * [A] 一つの Mod のデバッグ作業を念頭に置いて実装したため複数は未対応です。 プラグインIDを配列にすれば多分 foreach で回せると思いますが、対応はしません
* [Q] ゲーム実行中にプラグインIDの変更を行ったが反映されない
  * [A] Elin 起動時にしか監視処理登録を行いません。 随時監視対象の変更を追加するメリットと実装の面倒くささを考えると、今後もこの点に対する拡張はないかと
* [Q] デバッグ中の場合に読み込まれたらどうなるの？
* [Q] Mod 側で生成されたリソースはどうなるの？
  * [A] しらん！
